# 2. Upgrading ( Windows )

This is the simple guide for checking and updating:

- `n8n` in Docker
- the community `n8n-mcp` package used by `n8n_docs`

Important:

- `n8n_live` does not need a separate upgrade step
- the official MCP is part of your `n8n` instance, so updating `n8n` updates that too
- use this routine about once a month

## 1. Choose Your Community MCP Style

Pick one `n8n_docs` style and keep it.

### Option A: Automatic freshness

This is the default in this guide because it is the simplest setup for most people:

```toml
[mcp_servers.n8n_docs]
command = "npx"
args = ["n8n-mcp@latest"]
env = { MCP_MODE = "stdio", LOG_LEVEL = "error", DISABLE_CONSOLE_OUTPUT = "true" }
```

If you use the Windows path version, the args stay the same:

```toml
[mcp_servers.n8n_docs]
command = "C:\\Program Files\\nodejs\\npx.cmd"
args = ["n8n-mcp@latest"]
env = { MCP_MODE = "stdio", LOG_LEVEL = "error", DISABLE_CONSOLE_OUTPUT = "true" }
```

Leave it like this if you want the newest release on each restart.

### Option B: Stable pin (optional instead of the default)

Use this if you want the same community MCP version every time:

```toml
[mcp_servers.n8n_docs]
command = "npx"
args = ["n8n-mcp@1.2.3"]
env = { MCP_MODE = "stdio", LOG_LEVEL = "error", DISABLE_CONSOLE_OUTPUT = "true" }
```

If you use the Windows path version, the args stay the same:

```toml
[mcp_servers.n8n_docs]
command = "C:\\Program Files\\nodejs\\npx.cmd"
args = ["n8n-mcp@1.2.3"]
env = { MCP_MODE = "stdio", LOG_LEVEL = "error", DISABLE_CONSOLE_OUTPUT = "true" }
```

When you want to upgrade a pinned version, change only the version number in `config.toml`, save the file, fully close Codex, and reopen it.

You do not need to switch between these two styles unless you want to change how updates work.

## 2. Check The Latest Versions

Open these pages:

1. n8n update guide: [https://docs.n8n.io/hosting/installation/updating/](https://docs.n8n.io/hosting/installation/updating/)
2. n8n release notes: [https://docs.n8n.io/release-notes/](https://docs.n8n.io/release-notes/)
3. `n8n-mcp` on npm: [https://www.npmjs.com/package/n8n-mcp](https://www.npmjs.com/package/n8n-mcp)
4. `n8n-mcp` releases: [https://github.com/czlonkowski/n8n-mcp/releases](https://github.com/czlonkowski/n8n-mcp/releases)

## 3. Check Your Current Local `n8n`

Run this in PowerShell:

```powershell
docker exec n8n n8n --version
```

If your local version is older than the current stable release, continue with the next step.

## 4. Update `n8n`

Pull the newest image in PowerShell:

```powershell
docker pull docker.n8n.io/n8nio/n8n:stable
```

Then recreate the container with the same volume and settings:

```powershell
docker stop n8n
docker rm n8n

docker run -d --name n8n -p 5678:5678 `
  -e GENERIC_TIMEZONE="Asia/Singapore" `
  -e TZ="Asia/Singapore" `
  -e N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true `
  -e N8N_RUNNERS_ENABLED=true `
  -v n8n_data:/home/node/.n8n `
  docker.n8n.io/n8nio/n8n:stable
```

If you already use `WEBHOOK_URL`, keep that same `-e WEBHOOK_URL="..."` line in the command above.

Your workflows, credentials, and settings stay in the `n8n_data` volume.

Verify it in PowerShell:

```powershell
docker exec n8n n8n --version
docker ps
```

## 5. Check The Latest Community MCP Version

Run this in PowerShell:

```powershell
npm view n8n-mcp version
```

## 6. Update The Community MCP

If you use `n8n-mcp@latest`:

1. keep the file as-is
2. fully close Codex
3. reopen Codex
4. open `Settings -> MCP servers`
5. confirm `n8n_docs` is healthy

If you use a pinned version:

1. edit only the version number in `config.toml`
2. save the file
3. fully close Codex
4. reopen Codex
5. open `Settings -> MCP servers`
6. confirm `n8n_docs` is healthy

## 7. Run A Quick Smoke Test

In Codex, run:

```text
Use n8n_live.

List workflows.
```

Then run:

```text
Use n8n_docs.

Find the smallest no-credentials workflow pattern that uses Manual Trigger and Set.
Do not create anything yet.
```

## Fast Summary

- `n8n` update = pull the latest Docker image and recreate the container
- official MCP update = happens when `n8n` updates
- community MCP update = either reopen Codex when using `n8n-mcp@latest`, or bump a pinned version when you want to upgrade manually
